# raw-taxonomy-router

## 目标

减少原始材料入库时的分类混乱，先判断“这是什么”，再决定“该放哪里”。

## 默认分类

- `intelligence`
- `news`
- `research`
- `announcements`
- `policy`
- `data`
- `assets`
- `methodology`
- `sop`

## 判断优先级

1. 看来源类型
2. 看内容形态
3. 看后续用途
4. 看是否属于长期方法资产

## 常见规则

- 热帖、情报、交易线索 -> `intelligence`
- 媒体资讯、快讯批次 -> `news`
- 研报、纪要 -> `research`
- 公司公告、交易所文件 -> `announcements`
- 政策文件、监管口径 -> `policy`
- JSON、CSV、结构化表格 -> `data`
- 图片、PDF、原始附件 -> `assets`
- 规则、模板、方法草稿 -> `methodology` 或 `sop`

## 输出建议

优先给这几类判断：

1. `应保留`
2. `应改写`
3. `应拆分`
4. `暂不入库`

如果一个文件同时跨多类，优先建议拆分，而不是强塞进单目录。

## 边界

1. 只负责分类与路由判断，不负责正式采集。
2. 不把业务结论写进 raw/source 层。
3. 不因为历史目录脏乱，就继续沿用错误分类。
