# wiki原始分类路由

## 目标

减少原始材料入库时的分类混乱，先判断“这是什么”，再决定“该放哪里”。

## 默认分类

- `情报`
- `新闻`
- `研报`
- `公告`
- `政策`
- `数据`
- `附件`
- `方法论`
- `SOP`

## 判断优先级

1. 看来源类型。
2. 看内容形态。
3. 看后续用途。
4. 看是否属于长期方法资产。

## 常见规则

- 热帖、情报、交易线索 -> `情报`
- 媒体资讯、快讯批次 -> `新闻`
- 研报、纪要 -> `研报`
- 公司公告、交易所文件 -> `公告`
- 政策文件、监管口径 -> `政策`
- JSON、CSV、结构化表格 -> `数据`
- 图片、PDF、原始附件 -> `附件`
- 规则、模板、方法草稿 -> `方法论` 或 `SOP`

## 输出要求

1. 给出推荐分类。
2. 说明分类理由。
3. 如果材料需要拆分，写明拆分方案。
4. 如果不建议入库，明确说明原因。
