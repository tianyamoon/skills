## 一、研究前定义

正式研究前，先把问题定义清楚。定义不清，后面所有勤奋都会变成噪音。

1. 定研究对象
   - 行业名称是什么
   - 具体到哪一层：上游、中游、下游、渠道、应用端
   - 具体到哪一类产品、技术路线、客户群
2. 定研究边界
   - 地理边界：全球、中国、美国、欧洲，还是单一区域
   - 时间边界：过去一年、过去五年、上轮周期到本轮周期
   - 样本边界：上市公司、未上市龙头、海外映射、替代路线
3. 定研究问题
   - 我在判断阶段
   - 我在判断谁能穿越周期
   - 我在判断是否过热或过冷
   - 我在判断当前是否适合布局、持有或回避
4. 定输出形式
   - 一页纸结论
   - 深度报告
   - 公司组比较
   - 历史映射卡片
   - 周更或月更跟踪板
5. 定主变量假设
   - 默认先列出 3-5 个最可能驱动行业变化的变量
   - 这些变量后续要被证据验证或推翻

## 二、样本选择方法

研究行业，不要只看龙头。最少要有一个完整样本框架。

1. 龙头样本
   - 看行业最强者，理解利润池、定价权和扩张节奏。
2. 挑战者样本
   - 看谁在试图抢份额，判断竞争是升级还是恶化。
3. 上游样本
   - 看原材料、设备、能源、关键零部件，理解瓶颈与 CapEx 节奏。
4. 下游样本
   - 看终端客户、渠道、集成商，验证真实需求是否成立。
5. 失败样本
   - 看上轮扩张中谁被淘汰，判断周期尾部常见死法。
6. 阴影资产
   - 看二线公司、非上市产能、海外玩家、替代技术路线，避免只看到聚光灯下的供给。
7. 旁证样本
   - 看同类但不同区域、不同行业但相似周期机制的样本，用于做历史映射。

默认至少覆盖以下五类：

1. 一个最强龙头
2. 一个二线追赶者
3. 一个关键上游
4. 一个关键下游
5. 一个失败或弱势样本

## 三、变量表怎么建

变量表不是装饰，而是研究的骨架。没有变量表，就很容易掉进叙事陷阱。

### 1. 变量表结构

每个变量至少要有六列：

1. 变量名称
2. 变量方向
   - 上行利多
   - 下行利空
   - 双刃剑
3. 频率
   - 日频、周频、月频、季频、年频
4. 数据来源
5. 当前状态
6. 对结论的权重

### 2. 必备变量组

1. 需求变量
   - 出货量、装机量、订单、招标、客户资本开支、渗透率、更新替换周期
2. 供给变量
   - 产能、良率、开工率、扩产公告、在建工程、进入者数量、退出者数量
3. 成本变量
   - 原材料、能源、人工、折旧、运输、融资成本
4. 价格变量
   - ASP、现货价、合约价、折扣、价差、价格领导者动作
5. 库存变量
   - 原材料库存、渠道库存、客户库存、库存天数、预付款和应收款变化
6. 资本周期变量
   - CapEx、融资、发债、增发、补贴、设备交期、土地电力资源
7. 财务质量变量
   - 毛利率、经营利润率、ROIC、自由现金流、资本周转、经营现金流匹配度
8. 估值与拥挤变量
   - 估值分位、持仓集中度、融资余额、成交结构、预期一致性
9. 结构力量变量
   - 政策、技术替代、监管、地缘分裂、国产替代、行业标准变化

### 3. 变量表的使用方法

1. 先找领先变量，再找同步变量，最后找滞后变量。
2. 先找可以反驳自己观点的变量，再找支持变量。
3. 每次写结论时，至少回到变量表检查一次：
   - 哪些变量已验证
   - 哪些变量还没验证
   - 哪些变量开始反向

## 四、证据收集顺序

顺序很重要。顺序错了，容易被故事带着走。

### 1. 第一层：先看价格和市场行为

1. 绝对收益和相对收益是否领先行业基本面变化
2. 龙头与二线的股价强弱是否分化
3. 上游是否先于下游启动或见顶
4. 估值扩张和盈利上修，谁在前谁在后

目的不是用股价代替研究，而是把股价当作“市场已经提前在交易什么”的线索。

### 2. 第二层：再看新闻、政策和产业事件

1. 新政策到底改变需求、供给，还是只改变情绪
2. 重大订单、事故、制裁、补贴、技术突破是否会改变产业斜率
3. 这些事件是一锤子催化，还是会改变未来三年的资本开支与利润池

### 3. 第三层：再看行业数据

1. 产销数据
2. 开工率、稼动率
3. 现货价和价差
4. 库存和交期
5. 招标、订单、装机、出海数据

这一层主要负责验证“产业里到底有没有真实变化”。

### 4. 第四层：再看公司财务和管理层表述

1. 收入、利润、现金流是否一致
2. 扩产、折旧、存货、预付款、应收款有没有异常
3. 管理层表述与实际资本动作是否一致
4. 龙头与二线在扩产和定价上是否同步

### 5. 第五层：最后看估值、持仓和拥挤度

1. 市场一致预期是否过满
2. 当前估值需要怎样的增长与回报率才能成立
3. 资金是否已经从核心资产扩散到边缘资产
4. 是否出现“基本面刚验证，估值却已交易到多年远期”的情况

## 五、阶段判断方法

默认把典型行业周期拆成九段。不是每个行业都会完整走完九段，但大多数热行业都会在其中某几段里反复切换。

### 1. 主题导入期

特征：

1. 故事先行，数据很少
2. 龙头先涨，边缘公司跟涨
3. 估值扩张显著快于盈利验证

问题：

1. 主题是不是只停留在概念
2. 最早能验证的产业数据是什么

### 2. 产业验证期

特征：

1. 首批订单、客户导入、量产节点开始出现
2. 龙头基本面开始跟上
3. 产业链中的关键瓶颈变得可见

问题：

1. 需求是真的还是渠道备货
2. 进入门槛是否足够高

### 3. 订单放量期

特征：

1. 收入、订单、产能利用率同步上行
2. 龙头相对二线优势扩大
3. 上游设备、材料、能源开始受益

问题：

1. 增长是一次性补库还是可持续放量
2. 龙头是否开始激进扩产

### 4. 主升浪扩散期

特征：

1. 行业景气从少数公司扩散到多数公司
2. 卖方覆盖和市场关注快速抬升
3. 估值和盈利一起上行

问题：

1. 是产业扩散，还是资金扩散
2. 哪些二线公司只是贝塔，哪些真有份额逻辑

### 5. 核心资产估值顶期

特征：

1. 龙头公司被长期化定价
2. 一致预期极高
3. 微小不及预期就会触发估值压缩

问题：

1. 价格里已经隐含了几年高增长
2. 当前回报率是否还能覆盖估值风险

### 6. 上游或边缘资产接力期

特征：

1. 资金从最核心龙头扩散到上游、二线、小市值
2. 逻辑从“最确定”切换到“弹性最大”
3. 题材广度扩大，质量要求下降

问题：

1. 这是产业链轮动，还是末期拥挤外溢
2. 上游涨的是短缺逻辑，还是纯估值补涨

### 7. 资本开支失控期

特征：

1. 扩产公告密集
2. 新进入者增多
3. 资本市场和债务市场愿意提供融资

问题：

1. 当前项目回报是基于稀缺利润还是正常利润
2. 供给会在什么时候集中释放

### 8. 价格战与去库存期

特征：

1. ASP 下行
2. 库存上升
3. 订单推迟、客户观望、盈利下修

问题：

1. 是阶段性去库，还是结构性过剩
2. 龙头能否用成本和现金流熬过出清

### 9. 出清与新龙头重估期

特征：

1. 弱者退出、并购、停产或被边缘化
2. 供给收缩开始快于需求恶化
3. 龙头重新获得份额和定价权

问题：

1. 是旧周期尾声，还是新周期起点
2. 哪些公司具备下一轮穿越能力

## 六、资本周期判断

资本周期是行业研究里最容易被忽略、也最容易致命的部分。

### 1. 判断逻辑

1. 高利润会吸引供给进入。
2. 供给进入如果快于需求兑现，未来利润会被自己毁掉。
3. 所以要看的不是“公司现在多赚钱”，而是“行业正在如何响应这份高利润”。

### 2. 重点观察信号

1. 扩产公告是否从龙头扩散到全行业
2. 新进入者是否增多
3. 扩产融资是否变得容易
4. 设备交期、土地、电力、原材料是否支持大规模复制
5. 管理层是否从谨慎转向激进
6. 行业内是否出现“先扩再说”的情绪
7. 资本市场是否愿意为远期产能提前买单
8. 产能释放节奏是否集中在同一时间窗

### 3. 什么时候扩产是好事

1. 龙头扩产但二线跟不上
2. 扩产建立在成本优势、客户锁单和资金实力之上
3. 行业进入门槛高，新增供给不能轻易复制

### 4. 什么时候扩产会变成未来压力

1. 所有人都在扩
2. 项目 IRR 建立在当前高价而非正常价
3. 融资环境宽松掩盖了真实回报约束
4. 需求验证仍弱，但供给已先行
5. 管理层开始用市值故事驱动 CapEx

## 七、估值与预期判断

好的行业，不一定对应好的买点。这里必须把“好行业”和“好投资”拆开。

### 1. 先问价格里写了什么

1. 当前估值对应几年的高增长
2. 当前估值对应怎样的利润率和 ROIC
3. 当前估值默认市场份额能提升多少
4. 当前估值是否默认竞争没有恶化

### 2. 再问哪些预期最脆弱

1. 需求不及预期
2. 供给提前释放
3. 价格下降快于成本下降
4. 龙头护城河没有想象中深
5. 高估值无法容忍任何执行波动

### 3. 预期校验的常用方法

1. 反推法
   - 从股价反推收入、利润率、资本回报和终值假设。
2. 对比法
   - 对比历史高景气阶段的估值与当前所处位置。
3. 横向法
   - 对比同一产业链上下游、国内外可比公司的估值与预期差异。
4. 拥挤法
   - 看持仓集中度、主题热度、资金扩散路径。

## 八、穿越公司识别

行业很好时，很多公司看起来都很好。真正要找的是可以穿越周期的公司。

### 1. 核心筛选框架

1. 成本位置是否领先
2. 资产负债表是否稳健
3. 自由现金流质量是否可靠
4. ROIC 是否高于资本成本且具备持续性
5. 客户黏性、切换成本、渠道能力是否真实存在
6. 管理层资本配置是否克制
7. 在行业下行时是否还能保住份额和现金流

### 2. 典型强者特征

1. 上行期不盲目扩产
2. 下行期有能力逆势投入或并购
3. 与客户关系深，订单质量更稳定
4. 报表里现金流和利润相互印证

### 3. 典型弱者特征

1. 高景气时依赖融资而非经营现金流
2. 订单结构差，靠低价抢份额
3. 一旦价格回落，利润和现金流同时塌陷

## 九、历史复盘与行业映射

历史映射不是为了硬找一个相似故事，而是为了找相似的机制。

### 1. 先找机制，再找故事

优先寻找下列相似点：

1. 渗透率抬升
2. 供给瓶颈
3. CapEx 扩张
4. 上游利润扩散
5. 价格战与出清

### 2. 行业映射时的默认动作

1. 找一轮成功周期
2. 找一轮失败周期
3. 找一个相似但不完全相同的旁证行业
4. 比较它们在以下节点上的顺序：
   - 主题启动
   - 订单验证
   - 扩产启动
   - 上游接力
   - 估值见顶
   - 去库出清

### 3. 映射时不要做的事

1. 不要只因为都是“新技术”就判断它们会走一样的路
2. 不要只比股价，不比供需和 CapEx
3. 不要强行复制历史估值倍数

## 十、输出结论格式

默认按下面格式输出，保证研究可以复查和复用。

### 1. 一句话结论

用一句话回答：

1. 行业处在什么阶段
2. 最重要的驱动是什么
3. 当前最大的机会或风险是什么

### 2. 事实

列出已经确认的事实，不带结论词：

1. 需求
2. 供给
3. 价格
4. 库存
5. CapEx
6. 财务表现
7. 估值与持仓

### 3. 判断

基于事实给出解释：

1. 阶段判断
2. 资本周期判断
3. 估值与预期判断
4. 龙头与二线分化判断

### 4. 待补点

明确缺口：

1. 哪些变量没有数据
2. 哪些结论仍需要季度验证
3. 哪些关键分歧还没解决

### 5. 下一步

明确接下来最值得做的动作：

1. 继续跟踪哪些变量
2. 补哪些公司或哪类数据
3. 下一次更新的触发条件是什么

## 十一、禁止动作

1. 不准只因为股价涨就认定产业趋势成立。
2. 不准先有结论再挑证据。
3. 不准只看龙头，不看失败样本和阴影资产。
4. 不准把短期高利润直接外推成长期高回报。
5. 不准把估值贵解释成“好公司就应该贵”而不做预期反推。
6. 不准把政策口号当成订单和利润。
7. 不准在没有变量表的情况下写阶段结论。
