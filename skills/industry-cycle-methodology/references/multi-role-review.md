## 一、何时启用

以下场景默认启用多角色交叉审议：

1. 估值极高但产业叙事很强
2. 板块短期涨幅很大
3. 市场分歧极大
4. 资本开支进入快速扩张阶段
5. 需要做重仓、长期跟踪或形成公开结论

## 二、角色设置

默认使用六个角色。每个角色都要独立回答问题，不能互相代替。

### R1 主持人 / 总研究员

职责：

1. 定义主问题
2. 汇总争议点
3. 在最后形成综合结论

必须回答：

1. 当前到底在争什么
2. 哪些变量会决定争议结果

### R2 产业分析师

职责：

1. 解释产业供需、瓶颈、订单、价格和库存
2. 判断行业所处阶段

必须回答：

1. 现在到底有没有真实产业变化
2. 哪条传导链最关键

### R3 资本周期审计员

职责：

1. 审查扩产、融资、进入者和产能释放
2. 检查高利润是否正在诱发未来过剩

必须回答：

1. 供给是不是在失控
2. 最危险的时间窗在哪里

### R4 估值预期审计员

职责：

1. 从价格反推市场预期
2. 判断当前赔率是否还合理

必须回答：

1. 当前价格里已经写了什么
2. 哪个预期最脆弱

### R5 空头检察官

职责：

1. 只负责找 thesis 最可能错的地方
2. 强制寻找反证和失败路径

必须回答：

1. 这个研究最可能错在哪
2. 最可能导致大回撤的触发器是什么

### R6 风险经理 / 组合经理

职责：

1. 把研究翻译成行动纪律
2. 判断应该等待、跟踪、参与还是回避

必须回答：

1. 当前是否值得下注
2. 如果参与，最重要的风控条件是什么

可选角色：

1. 历史映射员
   - 当前局面更像哪段历史
2. 公司质量审计员
   - 谁能穿越周期

## 三、审议流程

### 第一步：定义主问题

主持人先把问题收敛成一句话，例如：

1. 这个行业当前是真产业主升，还是资金末期扩散
2. 这个龙头当前是长期机会，还是高预期陷阱

### 第二步：每个角色独立出结论

每个角色按同一格式输出：

1. 一句话观点
2. 最强证据
3. 最弱环节
4. 最关键待验证变量

### 第三步：整理分歧矩阵

把所有角色的分歧汇总为三类：

1. 已形成共识
2. 存在明显冲突
3. 缺证据无法裁决

### 第四步：裁决

主持人不能简单投票，而要按变量的重要性裁决：

1. 哪些变量是决定性变量
2. 哪些变量只是噪音
3. 哪个分歧必须先补证

### 第五步：形成行动结论

最后只输出四类行动之一：

1. 继续深入
2. 建立跟踪
3. 可以参与但要设风控
4. 暂时回避

## 四、分歧矩阵

默认按下面格式汇总：

```markdown
## 分歧矩阵

### 共识

1. 
2. 

### 冲突

1. 产业分析师认为：
2. 资本周期审计员认为：
3. 估值预期审计员认为：
4. 空头检察官认为：

### 待补证

1. 
2. 
```

## 五、裁决规则

1. 有证据的反方，优先级高于没有证据的乐观叙事。
2. CapEx 风险没有审清之前，不允许把高利润外推太远。
3. 估值已极端时，哪怕产业逻辑没错，也要单独降低赔率评分。
4. 如果空头检察官提出了明确证伪路径，必须正面回应。
5. 如果风险经理无法给出参与条件，默认维持观察而不是强行下结论。

## 六、输出格式

最终输出至少包括：

1. 主问题
2. 每个角色的一句话观点
3. 共识点
4. 核心冲突点
5. 最终裁决
6. 下一步验证动作

## 七、常见失真

1. 所有角色都在重复同一个观点，没有真正对抗。
2. 空头角色流于形式，只说套话风险。
3. 估值角色只看 PE，不反推隐含假设。
4. 风险经理没有把观点转成行动纪律。
5. 主持人为了统一意见，提前压掉真正分歧。
