## 一、设计原则

这个技能保持一个总入口，但内部按任务拆成多个研究模块。目的不是把技能拆碎，而是让每次研究都能明确：

1. 先做什么
2. 后做什么
3. 哪些模块必须出现
4. 哪些模块只在争议较大时启用

## 二、内部模块清单

### M1 行业边界定义模块

用途：

1. 定义行业、环节、产品、客户、区域、时间边界
2. 防止“行业”这个词太大，导致研究对象漂移

交付：

1. 行业边界一句话定义
2. 样本范围
3. 研究问题列表

### M2 样本与产业链建模模块

用途：

1. 构建龙头、挑战者、上游、下游、失败样本、阴影资产
2. 识别利润池与产业链传导方向

交付：

1. 样本表
2. 产业链简图
3. 关键瓶颈环节

### M3 变量表与证据采集模块

用途：

1. 建立需求、供给、价格、库存、CapEx、财务、估值变量表
2. 给每个变量绑定频率、来源和权重

交付：

1. 核心变量表
2. 领先、同步、滞后变量分层
3. 关键证据缺口

### M4 阶段判断模块

用途：

1. 判断行业处在主题导入、产业验证、主升扩散、估值见顶、出清等哪一段
2. 区分产业变化和资金变化

交付：

1. 当前阶段判断
2. 前后阶段切换条件
3. 阶段误判风险

### M5 资本周期审计模块

用途：

1. 审计供给扩张是否正在埋下未来过剩
2. 检查 CapEx、融资、进入者和产能释放节奏

交付：

1. CapEx 风险判断
2. 供给反噬时间窗
3. 哪类公司最容易在下行期出问题

### M6 估值与预期反推模块

用途：

1. 从价格反推市场已写入的增长、利润率、回报率和份额假设
2. 检查当前是产业机会、交易机会，还是拥挤风险

交付：

1. 预期反推结论
2. 最脆弱预期变量
3. 高估、合理、低估的条件边界

### M7 穿越公司筛选模块

用途：

1. 在行业波动里筛出能穿越周期的公司
2. 把好行业和好公司拆开分析

交付：

1. 公司组质量排序
2. 强者与弱者分层
3. 关键财务与经营证据

### M8 历史映射模块

用途：

1. 用相似机制做历史对照
2. 寻找成功周期、失败周期和旁证行业

交付：

1. 映射对象列表
2. 相似机制与不同机制
3. 当前所处历史相位的候选判断

### M9 标准输出模块

用途：

1. 把研究结果写成固定格式
2. 让后续跟踪、比对、复盘成为可能

交付：

1. 按模板生成的结论卡或报告
2. 事实、判断、待补点、下一步

### M10 复盘与验证模块

用途：

1. 记录 thesis、证据、触发条件和事后结果
2. 形成可评估的案例库

交付：

1. 研究台账
2. 研究评分
3. 错误分类与修正动作

### M11 多角色交叉审议模块

用途：

1. 让多头、空头、资本周期、估值与风险经理互相挑战
2. 在高分歧题材中降低单线程偏差

交付：

1. 多角色审议纪要
2. 核心分歧矩阵
3. 最终综合判断

## 四、标准模块组合

### 1. 最小可用研究包

`M1 + M3 + M4 + M9`

适用：

1. 快速判断一个行业是否值得继续研究

### 2. 标准行业研究包

`M1 + M2 + M3 + M4 + M5 + M6 + M9`

适用：

1. 正式写一版行业研究结论

### 3. 争议审计研究包

`M1 + M2 + M3 + M4 + M5 + M6 + M11 + M9`

适用：

1. 热门赛道
2. 高估值板块
3. 分歧大的主题交易

### 4. 长期跟踪研究包

`M1 + M2 + M3 + M4 + M5 + M6 + M9 + M10`

适用：

1. 建立长期跟踪名单
2. 周更、月更、季度复盘

## 五、模块输出接口

为了让模块能拼装，默认按下面接口传递：

1. M1 输出边界定义，供 M2-M11 全部引用。
2. M2 输出样本框架，供 M3、M5、M7、M8 引用。
3. M3 输出变量表，供 M4、M5、M6、M10 引用。
4. M4 输出阶段判断，供 M6、M8、M9 引用。
5. M5 输出 CapEx 审计，供 M6、M7、M11 引用。
6. M6 输出预期校验，供 M9、M10、M11 引用。
7. M7 输出公司分层，供 M9 引用。
8. M8 输出历史映射，供 M9 引用。
9. M11 输出分歧矩阵，供 M9 和 M10 引用。
