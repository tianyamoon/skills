# source-to-thesis-extractor

## 目标

把原始材料转成高密度、可复用的观点资产，减少“只归档、不提炼”的沉没成本。

## 适用来源

- 新闻与快讯
- 公司公告
- 研报与纪要
- 调研、访谈、会议材料
- 知识库中的 raw/source 层

## 推荐输出

1. 一句话 thesis
2. 关键催化
3. 为什么重要
4. 影响哪些主题、公司或对象
5. 后续验证什么
6. 哪些判断仍待核

## 提炼规则

1. 先区分“事实”与“事实意味着什么”。
2. 不复述整段原文，优先压缩成判断框架。
3. 如果材料不足以形成可复用 thesis，要明确说没有，不硬凑。
4. 如果结论只在某个阶段有效，要写清楚阶段性。
5. 如果判断只建立在单一来源上，要把“仍需验证”写出来。

## 边界

1. 不把单一来源直接升级成强结论。
2. 不替代完整研究报告。
3. 目标是高密度提炼，不是全文改写。
