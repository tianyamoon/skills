# MCP 取数路由

## 一、优先级

技术选股的原始数据，优先走本机 `luna-stock-data-mcp`。

当前本机配置里已经存在这个 MCP 服务；但每个 turn 实际暴露出来的工具名可能不同，所以不要死记某个函数名。先看当前工具列表里有没有以下能力，再决定怎么调：

1. `get_index_kline`
2. `get_stock_detail`
3. 个股 `K线` 能力
4. 个股 / 指数的 `日线 / 周线 / 月线` 查询能力

如果当前 turn 根本没有暴露股票 MCP 工具，要明确写：

`当前 turn 未暴露股票 MCP 工具，技术结论待核`

不要伪装成已经使用了 MCP。

## 二、取数顺序

### 1. 单票技术审查

至少拿这些数据：

1. 个股日线 K 线
2. 个股周线 K 线
3. 个股月线 K 线
4. 个股详情

如果周线或月线端点不可用：

1. 先拿日线 K 线
2. 再用本地脚本聚合成周线和月线

### 2. 多票比较

至少统一这些口径：

1. 同一天截止
2. 同一复权方式
3. 同样的 `5日 / 20日 / 60日 / 距60日高点`
4. 同样的月线乖离率口径

### 3. 指数参照

需要解释风格或板块位置时，再补：

1. 指数 K 线
2. 细分指数或科创50等对照指数

## 三、字段要求

脚本和人工计算至少需要这些字段；如果 MCP 字段名不同，先做映射：

1. `date / 日期`
2. `open / 开盘`
3. `high / 最高`
4. `low / 最低`
5. `close / 收盘`
6. `volume / 成交量`
7. `amount / 成交额`
8. `turnover / 换手率`

只要 `close` 缺失，就不能做趋势结论。

## 四、复权和时间口径

1. 默认优先使用统一复权口径。
2. 月线乖离率按月 K 口径计算。
3. 若当月未收盘，用 `动态月线口径`，并在输出中明写。
4. 只要不同来源的截止日不一致，就先写清楚，不要直接混算。

## 五、异常处理

下面情况都要显式降级为 `待核`：

1. MCP 返回空数组
2. MCP 返回 `ok:true` 但没有有效 K 线
3. 字段漂移，无法确认哪个字段是收盘价
4. 周线和月线端点与日线聚合差异明显
5. 日线数据不足，算不出 `20日 / 60日`
6. 月线数据不足，算不出 `6月线 / 12月线`

## 六、脚本用法

当 MCP 已经返回 K 线 JSON 后，优先用本技能自带脚本统一计算：

```bash
python scripts/calc_technical_metrics.py --daily daily.json
```

如果你已经分别拿到了周线和月线，也可以直接传入：

```bash
python scripts/calc_technical_metrics.py --daily daily.json --weekly weekly.json --monthly monthly.json
```

这个脚本会输出：

1. 月线趋势
2. 周线趋势
3. `6月线 / 12月线` 乖离率
4. `5日 / 20日 / 60日 / YTD`
5. `距60日高点`
6. `量价比`
7. `支撑 / 压力 / 证伪位`
