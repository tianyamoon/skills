# 输出模板

## 模板 A：单票技术审查

适用于：

1. `这票技术上还能不能买`
2. `月线周线怎么样`
3. `乖离率算一下`
4. `图形走坏了吗`

输出顺序：

1. `一句话结论`
2. `数据口径`
3. `月线资格`
4. `周线资格`
5. `月线乖离率`
6. `日线执行层`
7. `量价与位置`
8. `执行条件`
9. `证伪条件`
10. `数据缺口`

每一项都要有逻辑，不要只贴 `强 / 弱 / 看多 / 看空`。

## 模板 B：多票技术比较

适用于：

1. `这几只哪个图形更好`
2. `按技术面排个序`
3. `周一从图形里挑一只`

输出顺序：

1. `一句话结论`
2. `技术排序`
3. 逐只展开：
   - `月线`
   - `周线`
   - `乖离率`
   - `日线强弱`
   - `为什么排在这个位置`
4. `执行与证伪`

## 必写字段

### 1. 数据口径

至少写清楚：

1. 截止日期
2. 复权方式
3. 使用的是日线、周线、月线还是动态月线
4. MCP 是否正常返回

### 2. 月线资格

说明：

1. 是否满足 `月线向上`
2. 依据是什么
3. 如果不合格，为什么只能观察

### 3. 周线资格

说明：

1. 是否满足 `周线向上`
2. 周线是否已经确认执行资格
3. 若未确认，要等什么

### 4. 月线乖离率

说明：

1. `6月线乖离率`
2. `12月线乖离率`
3. 是否触发高位排除纪律

### 5. 日线执行层

至少覆盖：

1. `5日 / 20日 / 60日`
2. `YTD`
3. `距60日高点`

### 6. 量价与位置

至少回答：

1. 是 `放量强化`、`缩量整理` 还是 `放量转弱`
2. 当前位置更像 `突破区`、`高位消化`、`回踩观察` 还是 `趋势转弱`

### 7. 执行与证伪

必须落到：

1. 什么情况下允许进攻
2. 什么情况下只能观察
3. 哪个价位或哪条结构失守就算技术证伪

## 缺数据时怎么写

统一写法：

1. `MCP待核`
2. `月线待核`
3. `周线待核`
4. `月线乖离率待核`
5. `量价待核`

不要用猜测补空白。
