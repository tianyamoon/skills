# 技术选股

用这个技能把“图形看着强不强”的模糊讨论，压成一套先取数、再算指标、最后给执行条件的技术审查流程。

默认边界：

1. 先有数据，再有图形结论。
2. 月线和周线是资格线，日线只负责执行。
3. 技术面不能覆盖产业方向、基本面和估值纪律。
4. MCP 数据拿不到、字段不全或口径漂移时，必须写 `待核`。

## 工作流程

### 第一步：先识别任务类型

1. `单票技术资格审查`
2. `多个个股技术对比`
3. `月周线与乖离率复核`
4. `执行位 / 证伪位 / 支撑压力`
5. `对既有候选做技术层复核`

### 第二步：按需读取参考文件

1. 默认先读 [references/mcp-routing.md](references/mcp-routing.md)。
2. 计算前读 [references/indicator-scope.md](references/indicator-scope.md)。
3. 输出前读 [references/output-template.md](references/output-template.md)。
4. 需要稳定计算时，用 [scripts/calc_technical_metrics.py](scripts/calc_technical_metrics.py) 对 MCP K 线结果做统一计算。

### 第三步：固定顺序

`MCP取数 -> 数据口径确认 -> 月线资格 -> 周线资格 -> 月线乖离率 -> 日线执行层 -> 量价结构 -> 执行条件 -> 证伪条件`

不要反过来。不要因为日线大涨就跳过月线和周线。

### 第四步：数据口径纪律

1. 优先使用 `luna-stock-data-mcp`。
2. 先找当前 turn 里真正暴露出来的股票数据工具名，再调用；不要假设固定函数名一定存在。
3. 至少拿到日线；要判断技术资格时，再补周线和月线。
4. 能拿前复权或统一复权口径时，优先统一。
5. 若当月月 K 未收盘，`6月线 / 12月线` 乖离率按 `动态月线口径` 计算并注明。
6. MCP 若返回空数组、`ok:true` 但无有效数据、或字段漂移，必须写 `MCP待核` 并切到备选行情源复核。

### 第五步：只覆盖当前版本指标

当前版本只覆盖这些技术指标和技术字段：

1. `月线趋势`
2. `周线趋势`
3. `6月线乖离率`
4. `12月线乖离率`
5. `5日 / 20日 / 60日` 强弱
6. `YTD` 强弱
7. `距60日高点`
8. `量价结构`
9. `支撑位 / 压力位`
10. `执行位 / 证伪位`

`MACD / RSI / 布林带 / 缠论 / 波浪` 不在当前版本默认范围内，除非用户明确要求扩展。

### 第六步：硬纪律

1. `月线向上` 才有中期资格。
2. `周线向上` 才能谈执行。
3. `6月线乖离率 > 60%`，原则上不新增主动买入。
4. `12月线乖离率 > 100%`，原则上不新增主动买入。
5. `6月线乖离率在 40%-60%`，只能等分歧承接或平台消化后再评估。
6. 日线强，不能覆盖月周线不合格。
7. 只要数据口径不完整，就明确写 `待核`，不要硬下结论。

### 第七步：输出纪律

1. 先给一句话结论。
2. 明确数据来源、日期、复权和周期口径。
3. 把 `事实`、`判断`、`执行` 分开。
4. 多票比较时，先给技术排序，再展开每只。
5. 结尾必须落到执行或证伪，不要只说“趋势不错”。

## 不要这样做

1. 不要用技术面替代产业逻辑。
2. 不要把单日大阳线写成趋势反转。
3. 不要遗漏复权和时间口径。
4. 不要手算复杂指标后不留过程；能脚本算就脚本算。
5. 不要把缺数据默认当利多。
