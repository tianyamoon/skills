# 自媒体公众号专业文章协作工作流

## 协作原则

自媒体/公众号专业写作不是一次性代写，而是“作者经验 + 外部资料 + 编辑结构 + 复盘沉淀”的循环。

用户是主理人：提供真实经验、判断口径、账户语境和最终取舍。

Codex 是研究编辑：负责查资料、提炼结构、写初稿、做审稿、管理版本和沉淀方法，同时尽量把 AI 辅助推进到接近成熟人类作者的创作效果，而不是停留在模板化生成。

默认目标读者：A 股市场里有进取心、愿意学习和复盘、具备一定阅读耐心与理解力的个人投资者。他们希望读到专业、有密度、能进入自己交易体系的内容。

## 标准节奏

### 1. 定题

先确认：

1. 文章写给谁。
2. 读者痛点是什么。
3. 文章要让读者相信什么。
4. 读者读完能做什么。

产物：

1. 标题方向。
2. 一句话主张。
3. 文章边界。

### 2. 取材

材料分四层：

1. 用户经验：账户、复盘、交易执行里的真实观察。
2. 项目素材：`01_daily_reviews`、`02_analysis`、`03_exports`、wiki 方法资产。
3. 外部资料：研究文章、机构文章、平台规范、学术论文。
4. 反例素材：常见误区、错误操作、失败场景。

要求：

1. 外部资料只做证据和启发，不替代作者主张。
2. 关键事实必须可追溯。
3. 资料和观点要分层记录。

### 3. 搭骨架

推荐骨架：

1. 痛点场景。
2. 概念命名。
3. 为什么重要。
4. 错误做法。
5. 正确管理方法。
6. 自查表或操作清单。
7. 结尾收束。

### 4. 写初稿

初稿目标：

1. 先让文章通顺。
2. 再让观点锋利。
3. 最后才追求句子漂亮。

写作约束：

1. 段落短。
2. 小标题有判断。
3. 不堆术语。
4. 不用夸张承诺。
5. 先把读者带进去，再讲方法。
6. 写出作者的取舍和现场感，避免像资料总结或通用答案。

### 5. 共改

用户优先反馈三件事：

1. 观点准不准。
2. 哪些地方不像自己的话。
3. 哪些经验或例子应该加进去。

Codex 根据反馈改三层：

1. 结构顺序。
2. 表达风格。
3. 证据和例子。

### 6. 定稿

定稿前输出：

1. 标题备选。
2. 摘要或导语。
3. 正文。
4. 参考来源。
5. 风险提示。
6. 可拆成短内容的金句或段落。

## 文件落点

1. 工作稿：`F:\股票\股票复盘\02_analysis\YYYY-MM-DD_主题_公众号草稿.md`
2. 发布候选：`F:\股票\股票复盘\03_exports\YYYY-MM-DD\主题_公众号发布候选.md`
3. 写作技能：`F:\股票\股票复盘\skills\zmt-gzh-zy\`
4. 长期方法资产：经确认后再同步到 `F:\wiki\trade-system`

## 版本命名

建议使用：

1. `v1_结构通顺版`
2. `v2_公众号表达版`
3. `v3_发布候选版`

每次迭代记录：

1. 本轮主要改了什么。
2. 还缺什么。
3. 下一轮优先处理什么。

## 质量门槛

一篇文章进入发布候选前，必须满足：

1. 一句话主张清晰。
2. 开头能抓住目标读者。
3. 至少有一个可执行方法或自查表。
4. 没有未经核验的确定性市场判断。
5. 不像资料拼贴。
6. 读完能留下一个明确概念或方法。
