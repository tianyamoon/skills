# 自媒体公众号专业文章标准

## 目标

优秀自媒体/公众号专业文章不是把材料排版成长文，而是完成一次“读者注意力交易”：

读者付出时间，文章交付一个清晰判断、一套可执行方法，或一次认知纠偏。

## 默认读者画像

默认面向 A 股市场里有进取心、愿意学习和复盘、具备一定阅读耐心与理解力的个人投资者。

他们通常不是完全新手，也不是只想看一句结论的人。他们愿意读一篇有密度的文章，但前提是文章要帮他们把市场现象、交易困惑或账户问题解释清楚，并给出能带回自己复盘体系里的方法。

写作时默认尊重读者智力，不用过度口水化；同时照顾阅读节奏，不把文章写成研究报告、课堂讲义或资料堆叠。

## 选题标准

好选题通常满足四项：

1. 有明确读者：知道这篇写给正在进阶的 A 股个人投资者、成熟交易者、持仓焦虑者，还是复盘体系建设者。
2. 有真实痛点：读者在账户、认知、情绪或执行上确实会遇到。
3. 有作者主张：文章不是资料汇编，而是能说出“我怎么看”。
4. 有行动出口：读者读完知道下一步该检查什么、改变什么或警惕什么。

不成熟选题常见问题：

1. 只有概念，没有场景。
2. 只有观点，没有证据。
3. 只有资料，没有作者取舍。
4. 只有情绪，没有方法。

## 标题标准

标题承担两件事：让目标读者愿意打开，并准确承诺正文价值。

常用方向：

1. 痛点型：指出读者正在经历但尚未命名的问题。
2. 反常识型：推翻一个常见但粗糙的理解。
3. 方法型：承诺一套读完能使用的框架。
4. 收藏型：提供清单、评分表、复盘模板或判断标准。
5. 系列型：把单篇文章放入长期能力建设里。

标题禁区：

1. 夸大收益或制造确定性。
2. 正文无法兑现标题承诺。
3. 用惊悚词替代真实洞察。
4. 为了吸睛牺牲准确性。

## 开头标准

公众号开头要尽快完成三件事：

1. 让读者发现“这说的是我”。
2. 命名一个他们长期感到模糊的问题。
3. 给出继续读下去的理由。

推荐开头模式：

1. 场景开头：先写一个真实交易困境。
2. 反常识开头：指出读者习惯性误判。
3. 问题命名：把模糊痛苦变成一个可讨论概念。
4. 结论前置：先给核心判断，再解释。

避免：

1. 从大背景绕太久。
2. 第一段堆术语。
3. 先引用名人名言。
4. 开头就像研究报告摘要。

## 结构标准

适合专业自媒体文章的结构：

1. 场景痛点：读者为什么被这个问题困住。
2. 概念命名：给问题一个清晰名字。
3. 机制解释：为什么这个问题会破坏账户。
4. 外部证据：研究、机构观点、案例或经验样本支持。
5. 管理方法：给出可执行动作。
6. 检查清单：让读者马上能自查。
7. 收束：用一句简洁判断留下记忆点。

## 表达标准

1. 短段落优先，移动端每段尽量只讲一个意思。
2. 多用交易场景承接抽象概念。
3. 小标题要有信息量，不只写“第一”“第二”。
4. 转场要自然，让读者知道为什么从这一节走到下一节。
5. 术语出现后要立刻解释，避免只对作者自己清楚。
6. 金句可以有，但必须从论证里长出来。
7. AI 辅助生成的文字要继续做作者化处理：补足真实语境、删掉空泛排比、增加判断取舍，让文章接近成熟人类作者的自然创作效果。

## 证据与合规标准

1. 区分事实、研究结论、个人经验和交易判断。
2. 使用网络资料时转述和归纳，不大段复制。
3. 涉及具体证券时避免暗示确定收益。
4. 不把未核验资讯、论坛观点或 AI 总结当作事实。
5. 公众号发布前保留参考来源清单。
6. AI 辅助内容要经过真人观点、经验和表达重写，避免低创作度拼贴。

## 审稿清单

发布前逐项检查：

1. 标题是否准确兑现正文。
2. 第一屏是否足够吸引目标读者。
3. 一句话主张是否清楚。
4. 每个小节是否只服务一个功能。
5. 是否有可执行方法，而不是只讲道理。
6. 是否把事实、推演、判断分开。
7. 是否删除了重复、空泛和自我感动的段落。
8. 是否保留风险提示和参考来源。
