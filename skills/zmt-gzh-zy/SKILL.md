# zmt-gzh-zy

## 定位

把项目里的复盘、研究和方法资产，转成可用于公众号发布的专业自媒体文章：有读者意识、有作者判断、有结构推进、有表达节奏，读完能留下一个清晰概念、一套方法或一次认知纠偏。

本技能服务写作与编辑，不负责自动发布，不负责制造未经验证的市场事实，也不把公众号表达反向当作事实真源。

## 默认读者

面向 A 股市场里有进取心、愿意学习和复盘、具备一定阅读耐心与理解力的个人投资者。

他们不满足于听消息、追热点或看结论，而是希望理解市场背后的结构，把行情观察、交易复盘、账户管理和认知方法沉淀成可复用能力。

## 先读什么

按任务选择参考文件：

1. 写公众号文章、标题、开头或审稿时，先读 `references/wechat-article-standards.md`。
2. 与用户长期协作、边写边沉淀写法时，读 `references/collaboration-workflow.md`。

## 工作流

1. 定位读者：明确这篇文章写给谁、解决什么痛点、读完让读者采取什么动作。
2. 提炼主张：用一句话写出文章核心判断，避免只堆资料。
3. 搭结构：优先使用“场景痛点 -> 概念命名 -> 机制解释 -> 管理方法 -> 检查清单 -> 收束金句”。
4. 写初稿：用短段落、强转场、少术语、真实场景承接抽象概念。
5. 审稿：检查标题承诺是否兑现、事实和判断是否分层、是否存在低创作度拼贴、是否有投资建议风险。
6. 交付：给出正文、标题备选、摘要、参考来源和下一版修改方向。

## 公众号口径

1. 文章不是研究报告搬运，要有明确作者视角和读者收益。
2. 开头必须快速建立“这和我有关”，不要从宏大背景空转。
3. 每个小节只承担一个功能：提出问题、解释原因、给方法、做提醒。
4. 股票相关内容必须区分事实、推演和个人复盘判断。
5. 涉及个股、账户、策略时，保留风险提示，不承诺收益。
6. 使用外部资料时做转述和归纳，避免大段复制。
7. AI 辅助研究、结构、润色和审稿时，要尽量逼近成熟人类作者的创作效果：有判断、有取舍、有现场感、有节奏、有自我修正，避免模板腔、报告腔和资料拼贴感。

## 默认产物

自媒体/公众号文章任务通常输出：

1. `标题备选`：3-8 个，分别偏情绪、方法、反常识、长期收藏。
2. `一句话主张`：文章最核心的判断。
3. `正文初稿`：可继续协作打磨。
4. `发布前检查`：标题、开头、结构、证据、合规、风险提示。
5. `参考来源`：只列真正使用过的来源。

## 文件落点

1. 写作工作稿优先放 `F:\股票\股票复盘\02_analysis\`。
2. 发布候选稿放 `F:\股票\股票复盘\03_exports\YYYY-MM-DD\`。
3. 写作方法、模板、协作规则沉淀在本技能或 `references/`，不要塞进 `AGENTS.md`。
4. 若文章形成长期方法资产，再按项目规则同步到 `F:\wiki\trade-system` 的方法或 SOP 层。

## 边界

1. 不自动发布到微信公众平台。
2. 不用未经核验的行情、账户或个股事实支撑强结论。
3. 不把公众号文章当作 `reviews/`、`raw/` 或交易记录真源。
4. 不为了“爆款感”制造恐慌、夸大收益或暗示确定性。
