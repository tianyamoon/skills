# 输出模板

默认按这里的结构输出。

每个条件都要写逻辑，不要只写标签。

## 模板 A：方向优先回答

适用于：

1. `周一该进攻哪个方向`
2. `现在只能选一个方向`
3. `AIDC 和半导体设备哪个更值得做`

输出顺序：

1. `一句话结论`
2. `方向总排序`
3. 主攻方向逻辑
4. 其他方向为什么不是当前首选
5. 如有需要，再映射个股
6. `执行条件`
7. `反方撤退条件`

每个方向必须覆盖这些字段：

1. `方向地位`
2. `可验证路径`
3. `经营验证`
4. `估值预期`
5. `资金属性`
6. `位置与拥挤`
7. `低位纪律`
8. `技术门槛`
9. `排除法`
10. `反方条件`
11. `执行条件`

## 模板 B：方向内选股回答

适用于：

1. `推荐3个股票`
2. `这个方向里选哪几个`
3. `把个股逻辑展开`

即使是在选股问题里，也必须先给方向排序。

然后每只个股统一用这四个字段：

1. `角色`
2. `为什么看`
3. `为什么不无脑买`
4. `触发 / 证伪`

`为什么不无脑买` 里要吸收估值、拥挤、现金流、纯度、位置、技术风险等限制。

## 模板 C：比较并排除

适用于用户比较多个方向、多个主题、多个逻辑时。

对每个未被选中的方向，至少解释：

1. 可验证路径缺了什么
2. 经营验证缺了什么
3. 估值是不是已经太满
4. 资金语言是不是错配
5. 技术门槛是不是没过
6. 是哪条排除法或证伪条件挡住了它

## 必须保留的收尾句

结尾必须落回类似这句话：

`这不是概念标签选择，而是对可验证路径、经营证据、估值预期、资金属性和技术门槛的综合选择。`

## 缺数据时怎么写

如果数据不够新，统一写：

1. `月周线待核`
2. `月线乖离率待核`
3. `估值分位待核`
4. `经营验证待核`

不要默认把缺失项当成利多。
