# 游资方法论

这个文件只用于执行和短周期风控。

重要边界：

1. 研究方法，不研究身份。
2. 研究买入标准、卖出标准、仓位规则、退潮纪律。
3. 不把席位、榜单、自然人传闻当成方法论主体。

## 核心顺序

统一按下面顺序理解：

`市场周期 -> 主线确认 -> 龙头辨识度 -> 标准买点 -> 标准卖点 -> 仓位纪律 -> 退潮处理`

## 市场周期划分

至少要分清：

1. `启动`
2. `发酵`
3. `高潮`
4. `分歧`
5. `退潮`

一般规则：

1. 早期允许试错仓。
2. 确认扩散后才允许放大仓位。
3. 退潮期优先降仓，严重时直接空仓。

## 使用游资语言前必须确认什么

至少确认：

1. 有没有主线地位
2. 有没有辨识度
3. 有没有容量和流动性
4. 梯队是否完整
5. 分歧后有没有承接

这些都没有，就不要把它写成短线龙头。

## 标准买点

下面这些只能作为执行触发器：

1. `首板试错`
2. `二板确认`
3. `分歧转一致`
4. `弱转强`
5. `平台突破`
6. `趋势回踩承接`

它们不能覆盖月线和周线的硬门槛。

## 标准卖点

下面这些属于降级或退出信号：

1. `不及预期`
2. `断板无修复`
3. `低开无承接`
4. `高位放量滞涨`
5. `梯队断裂`
6. `连续负反馈扩散`

## 仓位纪律

仓位要跟确认度走，不要跟情绪走：

1. `试错仓`：方向早期或证据未满
2. `确认加仓`：验证增强后才加
3. `退潮降仓`：环境变差先缩
4. `退潮空仓`：短线环境明显恶化时退出

短线逻辑失败后，不要因为故事还在，就用补仓来掩盖错误。

## 在主标准中的位置

游资方法论只属于：

1. `资金属性`
2. `技术门槛`
3. `排除法`
4. `执行条件`

不属于：

1. `经营验证`
2. `可验证路径`

## 对用户解释时的简版口径

如果需要一句话解释，可以说：

这里说的 `游资`，指的是一套短周期选股和交易方法，不是某个席位、某个名字、某种传闻身份。
