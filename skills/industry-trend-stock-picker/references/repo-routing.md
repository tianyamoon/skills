# 仓库路由

当任务需要和 `F:\股票\ai-industry-cycle-research` 仓库口径保持一致时，使用这个文件。

## 什么时候启用

出现以下情况时启用：

1. 当前工作区就是 `F:\股票\ai-industry-cycle-research`
2. 用户说“按当前仓库标准”
3. 回答必须对齐仓库里的方法论，而不是泛泛聊股票

## 仓库内默认阅读顺序

优先读：

1. `03_结果层/02_正式写作区/AI交易策略日报/产业趋势研究选股标准_第二版.md`
2. `00_总控台/产业趋势研究选股标准_外部方法论共性矩阵_第一版.md`
3. `00_总控台/产业趋势研究选股标准_游资方法论实填修正版_第一版.md`
4. 如果用户问当前项目口径，再读 `00_总控台/项目状态面板.md`

## 来源优先级

统一按下面优先级：

1. 当前仓库正式标准
2. 仓库状态文件
3. 正式公告、财报、核验过的市场数据
4. 历史会话记忆
5. 通用经验判断

如果来源冲突，优先使用更保守的说法，并把缺口写明。

## 在这个仓库里的写法纪律

1. 把 `事实`、`判断`、`执行` 分开。
2. 证据不完整时，明确写 `待核`、`待补`、`待确认`。
3. 不要用短线热度覆盖产业趋势判断。
4. 如果用户在修改标准本身，先回写仓库文档，再把新规则当成生效规则。
