# 产业趋势选股

用这个技能把泛泛的题材讨论，压成一套“先选方向、再选个股、最后定执行”的可执行选股流程。

默认立场：
1. 方向优先于个股。
2. 经营验证大于故事。
3. 趋势只是门槛，不是逻辑本身。
4. 游资方法论只用于执行和风控，不用于证明基本面。
5. 技术门槛必须显式调用 `$technical-stock-picker`，不要手工脑补。

## 工作流程

### 第一步：先判断当前任务类型
先识别用户到底在问什么：

1. `方向排序`
2. `方向内选股`
3. `方向与方向之间的比较`
4. `排除法 / 反方条件 / 风险审计`
5. `对既有日报、周更或标准输出做升级`

### 第二步：按需读取参考文件
按递进方式读取：

1. 默认先读 [references/selection-standard-v2.md](references/selection-standard-v2.md)。
2. 输出前读 [references/output-template.md](references/output-template.md)。
3. 需要解释规则为什么这样设计，或需要区分机构、私募、量化、游资语言时，读 [references/external-calibration.md](references/external-calibration.md)。
4. 需要处理短周期买点、卖点、仓位、退潮纪律时，读 [references/hot-money-methodology.md](references/hot-money-methodology.md)。
5. 需要与 `F:\股票\ai-industry-cycle-research` 仓库口径保持一致时，读 [references/repo-routing.md](references/repo-routing.md)。
6. 只要问题涉及 `月线 / 周线 / 月线乖离率 / 支撑压力 / 执行位 / 证伪位`，先显式调用 `$technical-stock-picker`。让它负责 `MCP 取 K 线 -> 指标计算 -> 技术资格判断`，再把返回结果纳入本技能的方向与个股结论。

### 第三步：按固定决策链重建逻辑
所有候选方向和个股，都按下面顺序判断：

`方向排序 -> 可验证路径 -> 经营验证 -> 估值预期 -> 资金属性 -> 技术门槛 -> 排除法 -> 反方条件 -> 执行条件`

其中 `技术门槛` 一环必须引用 `$technical-stock-picker` 的结论；没有技术结论时，不要把方向或个股写成“可主动进攻”。

不要因为某一只股票看起来强，就跳过方向层直接下结论。

### 第四步：保留九条不可删除的纪律
这九条必须完整保留：

1. `方向优先于个股`
2. `经营验证大于故事`
3. `高位强票，不无脑追，审慎评价`
4. `低位方向不能因为没涨就买`
5. `必须有反方条件`
6. `买的是可验证的路径，不是概念标签`
7. `月线向上`
8. `周线向上`
9. `月线乖离率不过大`

如果某个回答漏掉了这些纪律中的关键项，要明确说明逻辑不完整，而不是硬给确定结论。

### 第五步：把资金语言和执行语言分开
统一区分四种语言：

1. `机构主导`：看长期利润池、现金流、ROIC、估值预期、资本周期。
2. `私募主导`：看产业景气、公司弹性、风格风险、组合容错。
3. `量化主导`：看趋势、因子、拥挤、容量、模型失效。
4. `游资主导`：看情绪周期、主线地位、辨识度、标准买点、标准卖点、仓位纪律、退潮纪律。

不要混写。尤其注意：

1. 不要用游资语言证明经营验证。
2. 不要用机构语言解释超短追涨行为。
3. 不要把席位、身份传闻或市场故事写成买入理由。

### 第六步：先过硬性排除条件
出现下面情况时，方向或个股至少降级为 `观察`，必要时直接归入 `回避`：

1. 没有经营验证。
2. 只有概念标签，没有可验证路径。
3. 估值已经透支多年完美预期。
4. 月线或周线不满足主动买入资格。
5. 月线乖离率过大。
6. 低位但没有趋势、没有拐点、没有催化。
7. 高位但没有消化、没有承接、没有估值缓冲。
8. 只有短线热度，没有基本面支撑。
9. 游资方法论里的标准买点不成立。
10. 游资方法论里的标准卖点已经触发。
11. 资金属性和逻辑解释错配。
12. 重资产方向面临资本周期反噬或信用收缩。

对第 `4 / 5 / 9 / 10` 项，优先采用 `$technical-stock-picker` 的结果；如果 MCP 数据未核到，明确写 `待核`。

### 第七步：按标准结构输出答案
输出时遵守以下规则：

1. 必须先给方向排序，再给个股。
2. 每个字段都要写逻辑，不能只贴标签。
3. 每只个股都要锚定在某个方向和某个角色里。
4. 月线、周线、乖离率、执行位和证伪位，由 `$technical-stock-picker` 提供技术证据；本技能负责把技术证据与方向、经营、估值、资金属性整合到同一结论里。
5. 把 `事实`、`判断`、`执行` 分开写。
6. 只要问题依赖当前市场、图形或财务的最新状态，就先核验再下确定结论。

## 输出纪律

字段名称和顺序以 [references/output-template.md](references/output-template.md) 为准。

最小输出顺序：

1. `一句话结论`
2. `方向总排序`
3. `主攻方向逻辑`
4. `其他方向为什么不是当前主攻`
5. `个股逻辑`
6. `执行条件`
7. `反方撤退条件`

结尾必须落回一句话：这是在选 `可验证路径`，不是在选概念标签。

## 不要这样做

1. 不要先挑股票，再反推方向。
2. 不要把低位或便宜本身当成逻辑。
3. 不要把高动量本身当成自证。
4. 不要把席位、身份传闻当成方法论。
5. 不要因为方向热门，就省略反方条件。
6. 不要用日线强势覆盖月线和周线的硬门槛。
7. 不要隐藏数据缺口；该写 `待核`、`待补`、`待确认` 就直接写。
