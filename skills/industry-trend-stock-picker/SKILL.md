# 股票产业趋势选股

## 定位

把泛泛的题材讨论，压成一套“先选方向、再选个股、最后定执行”的可执行选股流程。

默认立场：

1. 方向优先于个股。
2. 经营验证大于故事。
3. 趋势只是门槛，不是逻辑本身。
4. 游资方法论只用于执行和风控，不用于证明基本面。
5. 技术门槛必须显式调用 `$技术选股票`，不要手工脑补。

## 工作流程

### 第一步：先判断任务类型

1. `方向排序`
2. `方向内选股`
3. `方向之间比较`
4. `排除法 / 反方条件 / 风险审计`
5. `对既有日报、周更或标准输出做升级`

### 第二步：按需读取参考文件

1. 默认先读 [references/selection-standard-v2.md](references/selection-standard-v2.md)。
2. 输出前读 [references/output-template.md](references/output-template.md)。
3. 需要解释规则设计或区分机构、私募、量化、游资语言时，读 [references/external-calibration.md](references/external-calibration.md)。
4. 需要处理短周期买卖点、仓位和退潮纪律时，读 [references/hot-money-methodology.md](references/hot-money-methodology.md)。
5. 需要与 `F:\股票\ai-industry-cycle-research` 口径保持一致时，读 [references/repo-routing.md](references/repo-routing.md)。
6. 只要问题涉及 `月线 / 周线 / 月线乖离率 / 支撑压力 / 执行位 / 证伪位`，先显式调用 `$技术选股票`。

### 第三步：按固定决策链重建逻辑

`方向排序 -> 可验证路径 -> 经营验证 -> 估值预期 -> 资金属性 -> 技术门槛 -> 排除法 -> 反方条件 -> 执行条件`

### 第四步：保留九条硬纪律

1. `方向优先于个股`
2. `经营验证大于故事`
3. `高位强票，不无脑追，审慎评价`
4. `低位方向不能因为没涨就买`
5. `必须有反方条件`
6. `买的是可验证路径，不是概念标签`
7. `月线向上`
8. `周线向上`
9. `月线乖离率不过大`

### 第五步：把资金语言和执行语言分开

统一区分四种语言：

1. `机构主导`
2. `私募主导`
3. `量化主导`
4. `游资主导`

### 第六步：先过硬性排除条件

出现下面情况时，方向或个股至少降级为 `观察`，必要时直接归入 `回避`：

1. 没有经营验证。
2. 只有概念标签，没有可验证路径。
3. 估值已经透支多年完美预期。
4. 月线或周线不满足主动买入资格。
5. 月线乖离率过大。
6. 低位但没有趋势、没有拐点、没有催化。
7. 高位但没有消化、没有承接、没有估值缓冲。
8. 只有短线热度，没有基本面支撑。
9. 游资方法论里的标准买点不成立。
10. 游资方法论里的标准卖点已经触发。
11. 资金属性和逻辑解释错配。
12. 重资产方向面临资本周期反噬或信用收缩。

## 输出纪律

最小输出顺序：

1. `一句话结论`
2. `方向总排序`
3. `主攻方向逻辑`
4. `其他方向为什么不是当前主攻`
5. `个股逻辑`
6. `执行条件`
7. `反方撤退条件`

## 不要这样做

1. 不要先挑股票，再反推方向。
2. 不要把低位或便宜本身当成逻辑。
3. 不要把高动量本身当成自证。
4. 不要把席位、身份传闻当成方法论。
5. 不要因为方向热门，就省略反方条件。
6. 不要用日线强势覆盖月线和周线的硬门槛。
7. 不要隐藏数据缺口，该写 `待核`、`待补`、`待确认` 就直接写。
