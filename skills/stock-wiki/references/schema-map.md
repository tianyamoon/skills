# 股票 Wiki 结构地图

## 目标

快速判断一条内容该落到哪里，以及哪一层才是真源。

## 真源优先级

1. `raw/`：原始证据真源
2. `reviews/`：正式复盘真源
3. `topics/`、`entities/stocks/`、`cycles/`、方法页：结构化沉淀层
4. `output/`：交付层，不反向充当真源

## 目录角色

- `raw/`
  放公告、新闻、盘前情报、研报、政策、数据、附件、方法原稿。
- `topics/`
  放题材、产业链、催化链、上下游映射。
- `entities/stocks/`
  放个股页，强调主营、题材归属、催化、风险和关联。
- `cycles/`
  放周期阶段、阶段切换和主线演变。
- `reviews/`
  放正式复盘、次日计划、风险提示和正式判断。
- `concepts/`
  放概念定义与框架要点。
- `strategies/`
  放策略原则、执行框架、账户层方法。
- `tactics/`
  放战术动作、具体打法、执行细则。
- `sop/`
  放流程、清单、规则。
- `output/`
  放对外交付件或整理件。
- `review/`
  旧实验层，新内容默认不写入。

## 常见路由

- 公告、快讯、研报、截图：先去 `raw/`
- 一个题材的长期沉淀：去 `topics/`
- 一只股票的业务和题材映射：去 `entities/stocks/`
- 当日正式复盘判断：去 `reviews/`
- 交易方法、规则、流程：去 `concepts/` / `strategies/` / `tactics/` / `sop/`
- 给外部看的整理稿：去 `output/`

## 冲突处理

如果不同页面说法冲突：

1. 先回到 `raw/` 找证据。
2. 正式复盘判断以 `reviews/` 为准。
3. `output/` 只引用，不裁决。
