# 巡检规则

## 重点检查项

1. 新内容是否误写进 `review/`
2. `output/` 是否被反向当真源引用
3. `raw/` 是否有重要材料未分发
4. 题材页和个股页是否缺双向链接
5. 是否存在无任何 `[[wikilink]]` 的孤点页
6. 是否存在指向不存在页面的失效链接
7. `index.md` 和 `log.md` 是否跟新增页面脱节
8. 是否把事实、语料、复盘判断混写在一起

## 巡检输出分级

- `高`
  真源错放、正式路径写错、失效主入口、重要页面断链
- `中`
  关键页面缺关联、raw 未分发、导航未更新
- `低`
  结构可优化、命名可统一、段落可补齐

## 修复顺序

1. 先修真源和路径
2. 再修链接和导航
3. 再修页面结构
4. 最后修命名和美观

## 巡检时不要做

1. 不先斩后奏大搬迁历史目录
2. 不因局部脏乱就重写整套框架
3. 不把“可能相关”自动修成“明确关联”
