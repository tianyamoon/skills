# 股票行业逻辑评分

## 定位

用这个技能判断一篇行业研究到底是扎实研究，还是只是叙事流畅。

默认立场：

1. 把`研究质量`和`结论方向`分开。
2. 评分针对逻辑链，不针对文风。
3. 优先找最弱的一环，不重复表扬显而易见的优点。

## 核心原则

把高质量研究视作一条`审计顺序`：

`阶段 -> 变量 -> 传导 -> 资本周期 -> 经济性 -> 估值 -> 证伪 -> 跟踪`

如果一份研究不能按这条顺序重建，它就还不配得出强结论。

## 工作流

### 第一步：定义被评分对象

先识别：

1. 这份文档试图回答什么问题。
2. 它是行业 thesis、公司 thesis、桥接层卡片，还是正式总结。
3. 它使用的时间尺度是什么。
4. 它明确不回答什么。

如果文档边界不清，先指出这一点，再进入细项评分。

### 第二步：重建 thesis 逻辑链

按下面顺序重写论证链：

1. `事实`
2. `变量`
3. `传导`
4. `资本周期`
5. `经济性`
6. `估值 / 预期`
7. `验证 / 证伪`

如果无法重建这条链，本身就是一个结论。

### 第三步：给 10 个维度打分

使用以下权重：

| 维度 | 权重 |
|---|---:|
| 问题定义与边界 | 8 |
| 结构性力量与宏观背景 | 7 |
| 核心变量拆解 | 15 |
| 传导链完整度 | 12 |
| 供给 / CapEx / 资本周期审计 | 12 |
| 钱包份额 / 利润池 / 商业经济性 | 12 |
| 阶段定位与历史映射 | 10 |
| 估值 / 预期 / 拥挤度 | 12 |
| 证据质量与证伪能力 | 7 |
| 风险、跟踪点与下一步验证 | 5 |

### 第四步：应用硬性上限

即使文风很好，也要应用这些封顶规则：

1. 没有阶段判断：总分封顶 `75`。
2. 没有估值、预期或拥挤度讨论：总分封顶 `72`。
3. 重 CapEx thesis 但没有供给或 CapEx 审计：总分封顶 `68`。
4. 没有日期、来源或单位纪律：总分封顶 `60`。
5. 没有反证证据或主要风险：总分封顶 `78`。

### 第五步：诊断真实缺口

不要只说“还需要更多数据”，而要明确缺的是哪一类：

1. `缺阶段`
2. `缺变量`
3. `缺传导`
4. `缺供给 / CapEx 审计`
5. `缺经济性`
6. `缺估值`
7. `缺证据纪律`
8. `缺证伪`
9. `缺跟踪计划`

### 第六步：给修复项排优先级

默认修复顺序：

1. 补阶段判断。
2. 补主导变量。
3. 补传导链。
4. 补供给 / CapEx / 资本周期审计。
5. 补估值 / 预期 / 拥挤度。
6. 补证伪条件和跟踪点。

## 高分研究长什么样

1. 一开始就说明我们处在周期什么位置。
2. 能把需求、供给和资金条件分开。
3. 知道哪一层拿收入，哪一层拿利润。
4. 在外推增长前先审 CapEx 和资本回报。
5. 能区分“行业逻辑对”与“股票此刻值得买”。
6. 明确列出什么会证伪 thesis。

## 低分研究长什么样

1. 先有结论，再回填故事。
2. 列催化剂，但不排序变量。
3. 把题材热度误当成产业验证。
4. 讲 TAM，不讲利润池。
5. 忽略估值和拥挤度。
6. 发出去之后没有跟踪维护计划。

## 输出格式

默认使用以下结构：

1. `一句话结论`
2. `总分 + 等级`
3. `分维度评分表`
4. `前三个优点`
5. `前三个缺点`
6. `封顶项`
7. `最急需补的证据`
8. `最可能打破 thesis 的因素`
9. `下一个 5-10 分怎么拿`

## 语气与纪律

1. 直接，但不要把刻薄误当严谨。
2. 事实和判断分开写。
3. 如果证据不足，写 `待补`、`待核` 或 `待确认`，不要硬猜。
4. 如果 thesis 好但股票贵，要明确写出来。
5. 如果 thesis 差但文笔顺，不要奖励展示质量。

## 不要这样做

1. 不要因为文档堆了很多热词就给高分。
2. 不要让漂亮排版掩盖脆弱的变量链。
3. 不要把股价强势当成基本面验证。
4. 不要在不检查 CapEx、供给和现金流的前提下给重资产 thesis 打分。
5. 除非用户明确要求，不要把评分审计变成全文重写。
