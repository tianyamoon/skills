# wiki-link-repair

## 目标

修复知识页面之间的断链问题，让 wiki 从孤点页面恢复成可导航、可追踪的关系网络。

## 优先修复的关系

1. 主题 -> 主题
2. 主题 -> 实体
3. 实体 -> 主题
4. 复盘/总结 -> 主题或实体
5. 方法/SOP -> 相关页面

## 修复规则

1. 先补高价值链接，不追求一次全连满。
2. 优先补强关系，不是见到同名词就互链。
3. 能双向链接的尽量双向。
4. 链接要服务检索、导航和复用，而不只是让图谱好看。
5. 如果页面还不稳定，先标成待补链候选，不硬写死。

## 推荐顺序

1. 找出孤点页
2. 找出最强关系来源
3. 先补主干页
4. 再补细分页
5. 最后补方法页与复盘页引用

## 边界

1. 不因为同名词出现就机械加链接。
2. 不为了凑图谱新建空页面。
3. 重点是高质量连接，不是连接数量。
